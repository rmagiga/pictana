# 17. 画面フロー

## 17.1 起動フロー

```text
[Splash]
   ↓
[OS既定画像フォルダ検出]
   ├ 検出成功 → [Gallery Grid] (既定フォルダの画像表示)
   └ 検出失敗 → [Storage Selection]
```

初回起動でもOS既定の画像フォルダ（Android: DCIM/Camera等、Windows: ピクチャ）を自動検出し、即座にギャラリーを表示する。

## 17.2 通常フロー

```text
[Gallery Grid]
   ├ 画像タップ → [Image Viewer]
   ├ フォルダ変更 → [Storage Selection] → [Folder Browser] → [Gallery Grid]
   └ 設定 → [Settings]
```

## 17.3 プラットフォーム別フォルダ選択

```text
Android:
  [Storage Selection] → [Folder Browser (S03)] → [Gallery Grid]

Windows:
  [Storage Selection] → [OS標準フォルダ選択ダイアログ] → [Gallery Grid]
```

WindowsではFolder Browser (S03) を表示せず、OS標準ダイアログで完結する。

## 17.4 USB切断時

```text
全画面共通 → インラインバナー通知 (画面遷移なし)
```

画面遷移は発生しない。現在の画面上部にバナーを表示し、バックグラウンドで自動リトライする。

## 18.3 Gallery Grid

```text
+---------------------------------------+
| ← Camera       [🔍][↕ 名前▲][⚙]     |
+---------------------------------------+

+-----+ +-----+ +-----+
|img1 | |img2 | |img3 |
+-----+ +-----+ +-----+

+-----+ +-----+ +-----+
|img4 | |img5 | |img6 |
+-----+ +-----+ +-----+
```

### ソートUI

- `[↕ 名前▲]`: タップでソート基準メニューを表示
- ソート基準: 名前 / 日付 / サイズ / 種類
- 再タップで昇順↔降順を切替
- デフォルト: 名前 昇順

### 検索UI

`[🔍]` タップで検索バーを展開する。

```text
+---------------------------------------+
| 🔍 検索...                     [×]    |
+---------------------------------------+
| 種類: [JPEG] [PNG] [GIF] [全て]       |
+---------------------------------------+

+-----+ +-----+ +-----+
|img1 | |img2 | |img3 |
+-----+ +-----+ +-----+
```

- テキスト入力でファイル名のインクリメンタル検索
- 種類フィルター（チップ）で画像形式を絞り込み
- 検索中もグリッドはリアルタイムに結果を反映

### レスポンシブグリッド

グリッド列数はデバイスの画面幅に応じて動的に変化する。

| デバイス | 列数目安 |
|---|---|
| スマートフォン縦 | 3〜4列 |
| スマートフォン横 | 5〜6列 |
| タブレット | 5〜8列 |
| Windows | 4〜10列 |

ユーザーはSettings (S06) から列数を変更可能。

## 18.4 Image Viewer

### Windows レイアウト

```text
+---------------------------------------+
| ← IMG_0001.jpg                  [⋮]   |
+---------------------------------------+
|                                       |
|  [◀]       IMAGE VIEW         [▶]    |
|                                       |
|          pinch zoom                   |
|          pan                          |
|          Ctrl+MouseWheel zoom         |
|                                       |
+---------------------------------------+
|              1 / 152                  |
+---------------------------------------+
```

- `[◀]` `[▶]`: 画面左右端に表示される矢印ボタン。クリックで前後の画像に移動する。
- マウスホバー時のみ表示（フェードイン/アウト）。
- キーボード `←` / `→` キーでも同様に移動可能。

### Android レイアウト

```text
+--------------------------------+
| ← IMG_0001.jpg          [⋮]    |
+--------------------------------+

         IMAGE VIEW

       pinch zoom
       pan
       swipe (方向は設定で変更可能)

----------------------------------

1 / 152
```

### 画像移動操作

| プラットフォーム | 操作 | 動作 |
|---|---|---|
| Windows | 矢印ボタン `[◀]` `[▶]` クリック | 前/次の画像に移動 |
| Windows | キーボード `←` / `→` | 前/次の画像に移動 |
| Windows | キーボード `PageUp` / `PageDown` | 前/次の画像に移動 |
| Android | 左右スワイプ（デフォルト） | 前/次の画像に移動 |
| Android | 上下スワイプ（設定で有効化） | 前/次の画像に移動 |

### Android スワイプ方向設定

- Settings (S06) で「スワイプ方向」を選択可能
- 選択肢: 「左右」（デフォルト）/ 「上下」/ 「両方」
- 「上下」選択時: 上スワイプ = 次の画像、下スワイプ = 前の画像
- 「両方」選択時: 左右・上下どちらでも画像移動可能
- ズーム中はスワイプによる画像移動を無効にする（既存動作と同様）

### Windows 矢印ボタン仕様

- 画面左端中央に `[◀]`、右端中央に `[▶]` を配置
- マウスが画面上にある時のみ表示（フェードイン/アウト、200ms）
- 最初の画像では `[◀]` を非表示、最後の画像では `[▶]` を非表示
- ボタンサイズ: 48x48dp、半透明背景（Colors.black54）
- ズーム中も矢印ボタンは表示・操作可能

### `[⋮]` メニュー

```text
┌──────────────────┐
│ 📋 画像情報       │
└──────────────────┘
```

### 画像情報パネル（ボトムシート）

```text
+---------------------------------------+
| 画像情報                         [×]  |
+---------------------------------------+
| ファイル名    IMG_0001.jpg            |
| サイズ        4.2 MB                  |
| 解像度        4032 × 3024            |
| 形式          JPEG                    |
| 撮影日時      2026/05/10 14:30       |
| 更新日時      2026/05/10 14:30       |
+---------------------------------------+
```

## 18.5 USB切断インライン通知

画面遷移ではなく、現在の画面上部にバナーとして表示する。

```text
+------------------------------------------+
| ⚠ USB Storage が切断されました    [×]     |
|   自動的に再接続を試みています...          |
|   [ 別のフォルダを開く ]                   |
+------------------------------------------+
```

動作仕様:

- Gallery Grid / Image Viewer / Folder Browser の画面上部にバナー表示
- 自動リトライ（ポーリング）で再接続を検知
- 再接続時に自動でバナーを非表示＆内容を再読み込み
- 「別のフォルダを開く」で Storage Selection へ遷移可能
- キャッシュ済みサムネイルは切断中も表示継続

### スワイプ方向設定 (Android のみ)

| 値 | 動作 |
|---|---|
| 左右（デフォルト） | 左スワイプ = 次、右スワイプ = 前 |
| 上下 | 上スワイプ = 次、下スワイプ = 前 |
| 両方 | 左右・上下どちらでも画像移動可能 |

Windows ではこの設定項目は非表示（矢印ボタン＋キーボードで操作するため）。

# ナビゲーション仕様

## 戻る遷移チェーン

```text
Image Viewer → Gallery Grid → Folder Browser → Storage Selection
```

Storage Selection でバック操作 → アプリ終了。

## スクロール位置復元

- Gallery Grid: Image Viewerから戻った際、前回のスクロール位置を復元する
- Folder Browser: Gallery Gridから戻った際、前回のスクロール位置を復元する

## Androidバックジェスチャー

各画面で戻る遷移チェーンに従う。

# ローディング状態

## フォルダスキャン中

- スケルトンUI: グリッド形状のプレースホルダーを先に表示
- インクリメンタル表示: 全件ロード完了を待たず、取得済み分から即座に表示

## サムネイル生成中

- 個別セルにファイル形式アイコンのプレースホルダーを表示
- サムネイル生成完了後、フェードインで差し替え

## 画像ビューア読み込み中

- 低解像度サムネイルを先に表示し、原寸デコード完了後に差し替え
